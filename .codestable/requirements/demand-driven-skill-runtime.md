# 按当前动作加载规则和证据

## 用户故事

- 作为处理小型明确改动的开发者，我希望 CodeStable 的流程成本跟任务风险相匹配，而不是每次都带上完整的大型流程。
- 作为需要跨会话继续工作的开发者，我希望系统只保存恢复和质量门真正需要的证据，不必反复整理同一份验证结果。
- 作为独立审查代码的人，我希望拿到与当前变更严格对应的必要上下文，同时保留首次独立审查和输入漂移保护。
- 作为维护 CodeStable 的人，我希望每次流程调整都能用真实成本与质量结果判断是否值得推广，而不是只看文档变短。

## 为什么需要

CodeStable 已能按风险选择不同流程，但一次具体动作仍可能携带大量暂时用不到的规则、上下文和重复证据。小任务因此承担了接近大任务的成本，跨会话和跨审查者时还会重复整理相同材料。

## 怎么解决

系统根据当前动作装配必要规则和输入，只保存后续恢复、审查和决策真正需要的证据。新的装配方式先在可测量的小范围流程中验证，确认成本下降且质量不回退后，再逐步扩展。

## 边界

- 不以减少上下文为理由跳过测试、独立审查、人工确认或其他质量门。
- 不根据模型名称或推理档位选择更宽松的流程。
- 不改变默认 worktree、branch 或 checkout 策略。
- 不删除历史产物，也不强制尚未通过验证的流程迁移。
- 不采集或上传后台遥测；测量由维护者明确启动并保留可审计结果。
